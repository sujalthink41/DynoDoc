# DynoDoc v1 — API & Contracts

> The HTTP/SSE API surface **and** the `shared/contracts` port interfaces (the Protocols every adapter implements).
> Designed to **scale** (stateless, cursor-paginated, async for heavy work), be **testable** (thin routers + ports + fakes), and follow standard REST best practice.

# Part A — API Design

## A.1 Principles
1. **Stateless** — no server-side request affinity; all state in Postgres/Redis. Scale API pods horizontally.
2. **Resource-oriented REST** — plural nouns, correct verbs, correct status codes.
3. **Thin routers** — an endpoint only: validate (Pydantic) → call an application service → map result to a response DTO. **Zero business logic in `entrypoints/http`.** This is what keeps it testable and keeps logic in the domain.
4. **Async for heavy work** — anything AI-driven returns **`202 Accepted` + a job**, with progress via **SSE**. Nothing slow blocks a request.
5. **Consistent contracts** — one error shape, one pagination shape, one job shape, everywhere.
6. **Idempotent writes** — side-effecting POSTs accept an `Idempotency-Key` (ties to the points ledger's idempotency).
7. **OpenAPI is the source of truth** — auto-generated by FastAPI; drives the typed frontend client **and** contract/fuzz tests (`schemathesis`).
8. **DTOs at the boundary** — request/response models never expose ORM entities; domain stays decoupled.

## A.2 Conventions

| Concern | Decision |
|---------|----------|
| **Base path / versioning** | `/api/v1`. URI versioning. Within `v1` only **additive** changes; breaking changes → `/v2`. |
| **Auth** | Google OAuth → **httpOnly, Secure session cookie**; `SameSite=Lax`. **CSRF token** required on mutating requests. (Bearer-token API access deferred.) |
| **JSON casing** | `snake_case` end-to-end (matches Pydantic + DB). One convention, no surprises. |
| **IDs / time** | `uuid` strings; all timestamps **ISO-8601 UTC** (`...Z`). |
| **Errors** | **RFC 9457 `application/problem+json`** — `{type, title, status, detail, instance, errors[]}`. One shape for everything. |
| **Pagination** | **Cursor-based** (opaque cursor), `?limit=&cursor=`. No offset (doesn't scale). Response carries `page.next_cursor` + `page.has_more`. |
| **Long operations** | `202 Accepted` + `Location: /api/v1/jobs/{id}` + body `{job_id, status}`. Live progress via SSE; or poll the job. |
| **Idempotency** | `Idempotency-Key` header on side-effecting POSTs (submissions, course/lesson generation, daily responses). Replays return the original result. |
| **Rate limiting** | Per-user via Redis; `429` + `Retry-After`. |
| **Tracing** | `X-Request-ID` accepted/echoed; propagated to logs + workers for correlation. |
| **Status codes** | `200/201/202/204`, `400/401/403/404/409/422/429`, `5xx`. `409` for idempotency/version conflicts. |

## A.3 Standard Response Shapes

**Error (RFC 9457):**
```json
{ "type": "https://dynodoc.app/errors/validation", "title": "Validation failed",
  "status": 422, "detail": "goal must not be empty", "instance": "/api/v1/intake",
  "errors": [{ "field": "goal", "message": "must not be empty" }] }
```
**Collection (cursor pagination):**
```json
{ "data": [ /* items */ ], "page": { "next_cursor": "eyJpZCI6...", "has_more": true } }
```
**Job (async):**
```json
{ "job_id": "8f1c...", "status": "running", "progress": { "step": "writing_lecture", "message": "Writing Lecture 1…" } }
```
> Single resources are returned **bare** (not wrapped). Only collections use the `data`/`page` envelope.

## A.4 SSE Conventions (generation status + tutor streaming)
- `Content-Type: text/event-stream`; **named events**: `status`, `token`, `done`, `error`.
- Each frame has an `id:` → clients resume with `Last-Event-ID` after a drop.
- Periodic heartbeat comment (`: ping`) keeps proxies from closing the connection.
- Example (generation): `status` frames stream `Researching… → Writing Lecture 1… → ready`, then `done`.

# Part B — The Contracts (`shared/contracts`)

Ports are **`typing.Protocol`** interfaces (structural typing → adapters don't inherit, `mypy` enforces conformance). Two tiers:
- **Foundational ports** — generic technical capabilities.
- **Capability ports** — intent-revealing, ADK-backed (keeps ADK contained per ADR-003).

Every port ships with a **`Fake`** in `tests/fixtures/fakes/` used across unit/process/e2e tests.

## B.1 Foundational Ports

```python
# shared/contracts/llm.py
class TextGenerator(Protocol):
    async def generate(
        self, messages: list[Message], *,
        schema: type[BaseModel] | None = None,   # structured output → validated model
        model: ModelHint = ModelHint.DEFAULT,     # cheap vs strong (Strategy)
        temperature: float = 0.7,
    ) -> Generation: ...
    def stream(self, messages: list[Message], *, model: ModelHint = ...) -> AsyncIterator[str]: ...

# shared/contracts/embeddings.py
class Embedder(Protocol):
    dimension: int                                 # 1536 (text-embedding-3-small)
    async def embed(self, texts: list[str]) -> list[Vector]: ...

# shared/contracts/vector_store.py
class VectorStore(Protocol):
    async def upsert(self, records: list[VectorRecord]) -> None: ...
    async def search(self, query: Vector, *, owner_type: str, k: int,
                     filters: dict | None = None) -> list[VectorMatch]: ...
    async def delete(self, *, owner_type: str, owner_id: UUID) -> None: ...

# shared/contracts/search.py
class SearchProvider(Protocol):
    async def search(self, query: str, *, k: int = 5) -> list[SearchResult]: ...

# shared/contracts/sandbox.py
class CodeExecutor(Protocol):
    async def run(self, code: str, *, language: str, stdin: str | None = None,
                  timeout_s: float = 10, limits: ResourceLimits = ...) -> ExecutionResult: ...

# shared/contracts/events.py
class EventBus(Protocol):
    async def publish(self, events: Sequence[DomainEvent]) -> None: ...     # writes to outbox
    def subscribe(self, event_type: str, handler: EventHandler) -> None: ...

# shared/contracts/queue.py
class QueuePublisher(Protocol):
    async def enqueue(self, task: str, payload: dict, *,
                      idempotency_key: str | None = None) -> JobId: ...

# shared/contracts/clock.py
class Clock(Protocol):
    def now(self) -> datetime: ...      # injected → deterministic time in tests

# shared/contracts/unit_of_work.py
class UnitOfWork(Protocol):
    """Transaction boundary. Persists state + outbox events atomically."""
    async def __aenter__(self) -> "UnitOfWork": ...
    async def __aexit__(self, *exc) -> None: ...
    async def commit(self) -> None: ...
    async def rollback(self) -> None: ...
    # exposes domain repositories, e.g. self.courses, self.submissions, ...
```

## B.2 Capability Ports (ADK-backed, hide the agents)
Defined by the domains/processes that need them; implemented in `platform/agents/` using ADK + the foundational ports. Processes depend on **these**, never on ADK directly.

```python
# course domain / processes
class CurriculumPlanner(Protocol):
    async def next_intake_step(self, goal: str, history: list[QA]) -> IntakeStep: ...   # question(s) | profile
    async def plan_roadmap(self, profile: LearnerProfile) -> RoadmapDraft: ...

class LessonAuthor(Protocol):
    async def write_doc(self, topic: Topic, profile: LearnerProfile,
                        difficulty: Difficulty, variant: str) -> DocDraft: ...
    def stream_doc(self, ...) -> AsyncIterator[str]: ...

class ResourceCurator(Protocol):
    async def curate(self, topic: Topic) -> list[ReferenceDraft]: ...   # uses SearchProvider

class AssessmentAuthor(Protocol):
    async def build(self, lecture: LectureContext, profile: LearnerProfile) -> AssessmentSpec: ...

# assessment domain
class Grader(Protocol):
    async def grade_quiz(self, spec: QuizSpec, answers: dict) -> QuizResult: ...
    async def grade_code(self, spec: CodingSpec, submission: str,
                         execution: ExecutionResult) -> GradeResult: ...   # uses CodeExecutor

# tutoring domain
class Tutor(Protocol):
    def answer(self, question: str, context: list[VectorMatch],
               history: list[Message]) -> AsyncIterator[str]: ...          # uses Retriever + TextGenerator
```

## B.3 Repository Ports (per domain)
Each domain defines its own repository Protocol (e.g. `CourseRepository`, `SubmissionRepository`) in `domains/<x>/repository.py`. The `persistence.py` adapter implements it (the only domain file allowed to touch `platform/database`). Repositories are accessed through the `UnitOfWork` so a use case + its outbox events commit in one transaction.

## B.4 Why this is testable (the payoff)
- **Every port has a `Fake`** (e.g. `FakeTextGenerator` returns a fixed schema-valid `Generation`; `FakeClock` returns a frozen time; `FakeCodeExecutor` returns a scripted result).
- **Unit tests** wire services with fakes → no network, deterministic, millisecond-fast.
- **E2E tests** override FastAPI dependencies with fakes for AI/sandbox but keep real DB/Redis (Testcontainers):
```python
app.dependency_overrides[get_text_generator] = lambda: FakeTextGenerator(script=...)
# then drive the real HTTP endpoint via httpx ASGITransport and assert behavior + emitted events
```
- **Contract tests** run one shared suite against **both** the real adapter and its fake → guarantees Liskov substitution, so swapping a vendor (ADR-002/015/016) never breaks callers.
- **Quality** of AI outputs is measured by the eval suite (doc 04 B.4), not by brittle string assertions.

## B.5 Worked example — one slice, top to bottom (thin & testable)
```python
# entrypoints/http/routers/assessment.py  — THIN: no logic
@router.post("/assessments/{assessment_id}/submissions", status_code=202)
async def submit(assessment_id: UUID, body: SubmitRequest,
                 svc: SubmissionService = Depends(get_submission_service),
                 user: User = Depends(current_user),
                 idem: str | None = Header(None, alias="Idempotency-Key")) -> JobResponse:
    result = await svc.submit(user.id, assessment_id, body.to_command(), idempotency_key=idem)
    return JobResponse.from_domain(result)

# domains/assessment/service.py  — business logic, depends only on ports
class SubmissionService:
    def __init__(self, uow: UnitOfWork, grader: Grader, queue: QueuePublisher, clock: Clock): ...
    async def submit(self, user_id, assessment_id, cmd, *, idempotency_key) -> SubmitOutcome:
        # validate, persist submission (uow), enqueue code grading OR grade quiz inline,
        # publish assessment.* event in the SAME transaction (outbox). No HTTP, no SQL, no ADK here.
```
Unit-test `SubmissionService` with `FakeGrader/FakeQueue/FakeClock` + an in-memory UoW; e2e-test the route with fakes for AI and a real Postgres.

# Part C — API Decisions (ADRs)

- **AD-API-001 — URI versioning (`/api/v1`), additive-only within a version.** Predictable; old clients don't break.
- **AD-API-002 — RFC 9457 problem+json for all errors.** One machine-readable error contract across the whole API.
- **AD-API-003 — Cursor pagination, never offset.** Stable + scalable over large/secured datasets.
- **AD-API-004 — `202 + job + SSE` for all AI-heavy work.** No request blocks on generation/grading; horizontally scalable.
- **AD-API-005 — `Idempotency-Key` on side-effecting POSTs.** Safe retries; aligns with the points ledger's idempotency.
- **AD-API-006 — httpOnly session cookie + CSRF (Bearer deferred).** Secure default for a browser app.
- **AD-API-007 — Ports are `Protocol`s, each with a `Fake`.** Structural typing + fakes = swappable adapters and fast deterministic tests.
- **AD-API-008 — OpenAPI as source of truth.** Generates the typed frontend client and powers `schemathesis` contract/fuzz tests.

## C.1 Deferred
Webhooks/public API + Bearer tokens · GraphQL (REST is the right fit now) · API gateway/BFF split · response caching headers/ETags (add when read traffic warrants) · WebSockets (SSE suffices for v1).
